2. Heartbeat (keeps validator active + triggers sub-wallet auto-verify)
POST /api/validators/heartbeat
Content-Type: application/json

{
  "address":        "mxc1...",
  "batteryLevel":   85,
  "isCharging":     false,
  "epochSignature": {
    "epochNumber": 42,
    "signature":   "abcdef..."
  }
}

Success:

{
  "ok": true,
  "blockHeight": 4200,
  "timestamp": "2026-05-24T10:00:00.000Z",
  "openEpoch": {
    "epochNumber": 43,
    "blockHeight": 4300,
    "blockHash": "0xabc...",
    "signingWindowClosesAt": "2026-05-24T10:25:00.000Z"
  },
  "epochResult": { "ok": true, "nowFinalized": false }
}

Auto-Verify: Every heartbeat automatically re-checks any sub-wallets that have packageTier: null (not yet verified) against the smart contract. As soon as a sub-wallet gets activated on-chain, its packageTier is updated on the next heartbeat — no extra call needed from the app.

5. Sub-Wallet Lifecycle
1. Add sub-wallet (any address)
   └─ packageTier: null  →  shown as "Pending activation"
2. User buys $55 or $130 package on-chain with that wallet
3. Next validator heartbeat fires
   └─ server checks contract automatically
   └─ packageTier updated to "55" or "130"
4. Sub-wallet now earns epoch rewards (deposited to its frozenBalance)

The wallet app only needs to:

Call GET /api/validators/{addr}/sub-wallets after each heartbeat response
Show packageTier: null as pending, "55" / "130" as active
6. Epoch Rewards — How they land
When the main validator signs an epoch and quorum is reached, all activated wallets are rewarded automatically.

Wallet	Package	Reward per epoch	Lands in
Main validator	$55	~5 MC	frozenBalance
Main validator	$130	~12 MC	frozenBalance
Sub-wallet	$55	~5 MC	frozenBalance
Sub-wallet	$130	~12 MC	frozenBalance
Sub-wallet	none / null	0	—
Each wallet earns based on its own package, independently.

10. Complete Flow Summary
Register validator          POST /api/validators/register
  └─ must be activated on-chain first
Add sub-wallets             POST /api/validators/{addr}/sub-wallets
  └─ can add BEFORE activation — packageTier will be null
Send heartbeat every 8 min  POST /api/validators/heartbeat
  └─ include epoch signature
  └─ server auto-verifies any pending sub-wallets
Fetch sub-wallet list       GET  /api/validators/{addr}/sub-wallets
  └─ after heartbeat to see updated packageTier
Epoch rewards auto-credited → frozenBalance (main + all activated sub-wallets)
Check balances              GET  /api/admin/validators/{addr}/balances
                            GET  /api/admin/validators/{addr}/sub-wallets/{sub}/balances
Release frozen              POST /api/admin/validators/{addr}/release-balance
                            POST /api/admin/validators/{addr}/sub-wallets/{sub}/release-balance
Withdraw                    POST /api/transactions
