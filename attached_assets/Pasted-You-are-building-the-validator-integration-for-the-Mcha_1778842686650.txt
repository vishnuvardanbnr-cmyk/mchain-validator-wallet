HOW REWARDS WORK
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- Every active validator earns the same full MC reward each period — no heartbeat counting.
- The session stays alive as long as heartbeats keep arriving (every 15 min via background task).
- Each heartbeat resets a 3-hour countdown on the server.
- If no heartbeat arrives for 3 consecutive hours → server auto-pauses the validator → rewards stop.
- User can also manually pause/restart at any time.

STATUS MACHINE:
  pending → (admin approves) → active ←──────────┐
                                    │             │
                          (3h no heartbeat        │
                           OR user taps Pause)    │
                                    ↓             │
                                 paused ──(user restarts)─┘

inactive / banned = admin-only states, block all heartbeats.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
ENDPOINTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. REGISTER (once, after key generation)
POST /validators/register
{
  "address": "mxc1...",
  "ethAddress": "0x...",
  "publicKey": "<64+ char hex secp256k1 pubkey>",
  "deviceId": "optional",
  "moniker": "optional display name",
  "commissionRate": "10"
}
→ Validator is created as "pending". Poll status until "active" before starting heartbeats.

2. HEARTBEAT (background task, every 15 min, even when app is closed)
POST /validators/heartbeat
{
  "address": "mxc1...",
  "batteryLevel": 85,
  "isCharging": true,
  "activeMinutes": 15
}
Success 200: { "ok": true, "blockHeight": 12345, "timestamp": "..." }
Error 403 responses and how to handle them:
  - "validator_paused"              → show push notification, set local status = paused
  - "Validator is pending approval" → show "awaiting approval" UI
  - "Validator is inactive"         → show "contact support"
  - "Validator is banned"           → show banned message

3. PAUSE (user taps "Pause Validator")
POST /validators/pause
{ "address": "mxc1..." }
→ { "ok": true, "status": "paused" }
Call unregisterHeartbeatTask() before this.

4. RESTART (user taps "Restart" after any pause)
POST /validators/restart
{ "address": "mxc1..." }
→ { "ok": true, "status": "active", "sessionStartedAt": "..." }
Call registerHeartbeatTask() after this.

5. STATUS CHECK (on every app open)
GET /validators?status=all&limit=500
Filter by user's address. Show UI based on "status" field:
  active   → "Validating — earning rewards" + Pause button
  paused   → "Paused — not earning" + Restart button
  pending  → "Awaiting admin approval"
  inactive/banned → appropriate message

6. EARNINGS
GET /validators/:address/earnings
Returns total MC earned, reward history, delegation info.

BACKGROUND TASK IMPLEMENTATION (Expo)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

import * as BackgroundFetch from 'expo-background-fetch';
import * as TaskManager from 'expo-task-manager';
import AsyncStorage from '@react-native-async-storage/async-storage';
import * as Notifications from 'expo-notifications';

const HEARTBEAT_TASK = 'mchain-validator-heartbeat';

TaskManager.defineTask(HEARTBEAT_TASK, async () => {
  try {
    const address = await AsyncStorage.getItem('validatorAddress');
    if (!address) return BackgroundFetch.BackgroundFetchResult.NoData;

    const status = await AsyncStorage.getItem('validatorStatus');
    if (status === 'paused' || status === 'inactive' || status === 'banned') {
      return BackgroundFetch.BackgroundFetchResult.NoData;
    }

    const response = await fetch('https://chain.mvault.pro/api/validators/heartbeat', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ address, activeMinutes: 15 }),
    });

    if (response.status === 403) {
      const data = await response.json();
      if (data.error === 'validator_paused') {
        await AsyncStorage.setItem('validatorStatus', 'paused');
        await Notifications.scheduleNotificationAsync({
          content: {
            title: 'Validator Paused',
            body: 'Your phone was offline too long. Open the app to restart and keep earning.',
          },
          trigger: null,
        });
      }
      return BackgroundFetch.BackgroundFetchResult.Failed;
    }

    await AsyncStorage.setItem('validatorStatus', 'active');
    return BackgroundFetch.BackgroundFetchResult.NewData;
  } catch {
    return BackgroundFetch.BackgroundFetchResult.Failed;
  }
});

export async function registerHeartbeatTask() {
  await BackgroundFetch.registerTaskAsync(HEARTBEAT_TASK, {
    minimumInterval: 15 * 60, // 15 minutes
    stopOnTerminate: false,   // keep running on Android when app is closed
    startOnBoot: true,        // restart after phone reboot
  });
}

export async function unregisterHeartbeatTask() {
  await BackgroundFetch.unregisterTaskAsync(HEARTBEAT_TASK);
}

WHEN TO CALL:
- registerHeartbeatTask()   → after validator is approved and user taps Start
- unregisterHeartbeatTask() → when user taps Pause (before calling /validators/pause)
- On app launch: always re-register the task in case it was cleared (iOS/Android may remove it)
