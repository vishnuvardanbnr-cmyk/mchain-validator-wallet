ERC-20 Token Transfer
How it works
You call the token's smart contract transfer(address,uint256) function by sending a contract_call transaction to the contract address with ABI-encoded calldata.

Step 1 — Encode the calldata
import { ethers } from "ethers"
const ERC20_ABI = ["function transfer(address to, uint256 amount) returns (bool)"]
const iface = new ethers.Interface(ERC20_ABI)
const calldata = iface.encodeFunctionData("transfer", [
  "0xRecipientEthAddress...",    // recipient (0x format)
  "1000000000000000000"          // token amount in smallest unit (e.g. 1 token = 10^18)
])
// calldata = "0xa9059cbb000......"

Step 2 — Get nonce & sign
const from            = "mxc1abc..."              // your wallet
const to              = "mxc1contractAddress..."   // token contract address
const amount          = "0"                        // native MC sent = 0 (you're just calling the contract)
const nonce           = 5                          // from GET /api/accounts/{address}
const message = `MChain Transfer\nfrom: ${from}\nto: ${to}\namount: ${amount}\nnonce: ${nonce}`
const signature = await wallet.signMessage(message)

Step 3 — Submit
POST /api/transactions
Content-Type: application/json

{
  "fromAddress": "mxc1abc...",
  "toAddress":   "mxc1contractAddress...",
  "amount":      "0",
  "nonce":       5,
  "data":        "0xa9059cbb0000000000000000000000001234...0000000000000000000000000000000000000000000000000de0b6b3a7640000",
  "txType":      "contract_call",
  "signature":   "0x4a3f..."
}

Key differences vs native transfer
Native MC	ERC-20 Token
toAddress	Recipient wallet	Token contract address
amount	MC amount in Wei	"0" (no native coin sent)
data	not needed	ABI-encoded calldata
txType	"transfer" (default)	"contract_call"
Full example in ethers.js
import { ethers } from "ethers"
// 1. Encode calldata
const iface = new ethers.Interface(["function transfer(address,uint256) returns (bool)"])
const data = iface.encodeFunctionData("transfer", [
  recipientEthAddress,   // 0x...
  tokenAmountWei         // BigInt or string
])
// 2. Sign
const nonce = (await fetch(`/api/accounts/${fromAddress}`).then(r => r.json())).nonce
const message = `MChain Transfer\nfrom: ${fromAddress}\nto: ${tokenContractAddress}\namount: 0\nnonce: ${nonce}`
const signature = await wallet.signMessage(message)
// 3. Submit
await fetch("/api/transactions", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    fromAddress: fromAddress,
    toAddress:   tokenContractAddress,
    amount:      "0",
    nonce:       nonce,
    data:        data,
    txType:      "contract_call",
    signature:   signature
  })
})

The chain already decodes transfer(address,uint256) calldata in the explorer so the recipient and token amount show correctly in transaction history.
