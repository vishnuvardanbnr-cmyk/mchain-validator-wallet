Where "Could not coalesce error" likely comes from
This error almost always appears in wallets that use WalletConnect v1 or a custom JSON-RPC bridge. The most common causes:

Cause	Fix
wallet_addEthereumChain returns non-null (e.g. true) instead of null	Return null on success per EIP-3085
eth_sendTransaction returns the hash without "0x" prefix	Always return "0x" + txHash
Wallet calls eth_estimateGas and gets 0x0 back → coerces to 0 → fails	Respect the gasLimit already in the TX and skip estimation
Wallet tries to fetch nonce via eth_getTransactionCount with "pending" tag	MChain may not support "pending" tag — fall back to "latest"
Wallet calls eth_getBlockByNumber("latest", false) to check gas prices and can't parse the response	MChain blocks have custom fields (MXC bech32 miner address) — wallet must not crash on unknown fields
Summary for the wallet dev
The DApp sends standard EIP-1193 eth_sendTransaction with an explicit gasLimit. The wallet should:

Support wallet_addEthereumChain (EIP-3085) returning null on success
Pass gasLimit straight into the TX — do not call eth_estimateGas (returns 0 on MChain)
